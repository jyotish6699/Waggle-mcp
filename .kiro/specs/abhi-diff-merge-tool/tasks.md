# Implementation Plan: `.abhi` Diff/Merge Tool

## Overview

Harden the existing skeleton `diff_abhi_documents` / `merge_abhi_documents` functions in `src/waggle/abhi.py` into a production-quality field-level diff and three-way merge workflow. The work proceeds in six phases: data models, field-level diff engine, merge engine, boundary enforcement, CLI/MCP surface, and the test corpus.

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- Each task references specific requirements for traceability
- Checkpoints ensure incremental validation
- Property tests use [Hypothesis](https://hypothesis.readthedocs.io/) with `@settings(max_examples=100)` (5 for performance tests)
- `AbhiDiffResult` is retained for backward compatibility; `FieldLevelDiffResult` is the new canonical type for all new callers
- The `contradict` strategy is the default merge strategy; it creates synthetic `CONTRADICTS` edges rather than discarding either value
- All logic stays within `src/waggle/abhi.py` and `src/waggle/models.py` — no new top-level modules
